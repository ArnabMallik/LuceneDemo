Assignment 1 - Developer

Due: 1 February 2019 11:55pm

Goal

The goal of this assignment is to create, use, and modify a simple information retrieval system using Lucene. This will help you begin to understand how search engines work, and how documents and queries can be represented for information retrieval tasks. These representations will be valuable later as we discuss other tasks, including classification, clustering, and topic modeling.

Details

This assignment is to be done individually. You may discuss the project with your classmates, but the work you turn in should be your own.

Lucene is an open source project from Apache that consists of a set of Java classes that can easily be assembled to create a search engine that can be modified and customized. Note that Lucene has been ported to other languages. You may use a language other than Java, but no support from the instructors will be available for any language other than Java.

The Java API is available online and can be very helpful: https://lucene.apache.org/core/7_2_0/index.html

Setup

  1. If you are on a Mac, ensure that you have command line tools installed by running xcode-select --install in a terminal window.
  2. Make sure you have Java JDK 8 installed. This assignment was tested with java version 1.8.0_45. (Check by executing the command java -version in a terminal window.) You should see something like java version "1.8.0_45" Java(TM) SE Runtime Environment (build 1.8.0_45-b14) Java HotSpot(TM) 64-Bit Server VM (build 25.45-b02, mixed mode) We do not recommend Java 9 and have seen it cause problems. Java can be downloaded at https://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html.
  3. In a unix-like environment, you can use . setclasspath.sh while in your assignment directory to set your classpath; Windows users may need to use a different approach but can use setclasspath.sh as a reference. If you are familiar with a Java IDE, you may use that.
  4. Compile the provided MyStandardAnalyzer.java, MyIndexFiles.java, and MySearchFiles.java sources. They have been copied directly from the Lucene demos: https://lucene.apache.org/core/7_2_0/demo/overview-summary.html and produce identical results.
    1. To run the original indexing demo: java org.apache.lucene.demo.IndexFiles -docs documents/
    2. To run the original query demo: java org.apache.lucene.demo.SearchFiles
    3. To run our version of the indexing demo: java MyIndexFiles -docs documents/
    4. To run our version of the query demo: java MySearchFiles

Modifying the Demo

In the three sections below, you will modify the provided code in the provided MyStandardAnalyzer.java, MyIndexFiles.java, and MySearchFiles.java sources to change how Lucene processes documents and queries. Rather than starting "from scratch" in each section, make changes to your files incrementally; e.g., after you have made modifications affecting stopping, keep those modifications as you make further modifications affecting stemming and similarity. You should submit one set of files that includes all modifications.

Hint: When you modify files, it may be necessary to re-index your documents after compiling your sources in order to see those changes reflected at query time.
